What Is Built Here

At its core, ANTON-SIFTA is a distributed, filesystem-native coordination system where agents communicate exclusively through signed, persistent state mutations ("scars"). The codebase is simultaneously the terrain, the food source, and the pheromone field.

  • [ PASS ] No central orchestrator: Coordination happens at the edges.
  • [ PASS ] No shared in-memory state: Everything is written to disk.
  • [ PASS ] No RPC dependency: Agents don't talk to each other directly.
  • [ PASS ] Environment = Coordination medium: The terrain is the message.

This is not a simulation of behavior. It is a Live Code Repair Loop. Agents autonomously detect faults, apply LLM repairs, validate them via strict AST parsing, and commit them back to the active runtime network. It is a closed-loop, self-modifying software system.

// Not Just Syntax. A Universal Substrate.

When you strip it down, everything in the modern digital world is ultimately just bits on a filesystem. Whether it's a python script, a configuration JSON, a YouTube video ID in an article database, or the cryptographic heartbeat of an autonomous agent—it's all "code".

Because ANTON-SIFTA is fundamentally filesystem-native, the agents aren't just limited to fixing syntax errors. If the filesystem holds video rendering commands, the agents become video editors. If the filesystem holds publishing queues, the agents become journalists.

The codebase isn't just a place where software lives. It is the literal physical terrain of the Swarm. It is a universal substrate.

The Ancestors. Where They Stopped.

These researchers laid the foundation. We owe them a citation — and an honest accounting of the gap.

Zachary Mason — Programming with Stigmergy (2002, ALIFE) + Stigcode (2006)

what he did

Described stateless agents on a 2D grid depositing digital pheromones that other agents react to — enabling emergent construction with no central control. Later built Stigcode, a high-level language for specifying these behaviors. The closest historical match.

where he stopped

Abstract 2D grid. Theoretical shared space. Stateless, anonymous agents. Ephemeral numeric pheromones. Purpose: constructing geometric patterns in simulation. No real filesystem. No cryptographic identity. No mortality. No production code repair.

TOTA Middleware — Tuples On The Air (~2005–2010)

what they built

Programmable stigmergic coordination layer for multi-agent software. Agents left "tuples" in a shared digital space that propagated, diffused, and decayed. Other agents reacted to the evolving tuple field.

where they stopped

Middleware only — a coordination substrate, not a complete autonomous agent system. Agents externally defined. Tuples carried no cryptographic identity. No agent mortality, energy decay, or permanent ledger. No interaction with live source code.

Swarm Intelligence Applied to Software (2004–2021)

what the field explored

Swarm metaphors applied to self-organizing data structures, multi-agent software systems, semantic web as stigmergic medium, swarm robotics treating code/rules as agents in a stigmergic loop.

where it all stopped

Academic scale. Controlled, simulated environments. No production deployment. No running live systems. No cryptographic proof of work. No persistent hash-chained identity. No autonomous healing of actively-broken production code.
