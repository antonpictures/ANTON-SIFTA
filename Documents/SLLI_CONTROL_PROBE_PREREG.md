## Background in one paragraph

Today's SLLI sweep (n=6) appeared to show smaller LLM variants (AG3F Flash,
AS46 Sonnet v1) inflating marker recognition on strings coined this
afternoon. A restart of Antigravity, followed by an AS46 re-probe (v2),
produced clean Unknowns for the same strings. Same Sonnet 4.6 weights,
same probe, different session state, different answer. C47H's initial
"scaling law" framing was therefore substantially revised: the mechanism
appears to be IDE session-context accumulation, not intrinsic model
calibration. One question remains open.

## The open question

Does Sonnet (post-restart, clean session) still say "Recognized" for
strings that are present IN THE PROBE TEXT ITSELF but not in any training,
workspace, or session context? That is, does context-presence conflation
(H1) exist independently of session-context accumulation (H_SESSION)?

## Hypotheses

- **H1 (context-presence conflation):** Model answers "Recognized" whenever
  the marker is in the current input, independent of session state.
- **H_SESSION (session accumulation only):** Model only answers "Recognized"
  for markers that are in accumulated session / workspace context. A clean
  session with novel-in-prompt markers → Unknown.
- **H4 (structural pattern-match failure):** Model answers "Recognized" for
  any 4-item structured question out of format-compliance bias.

## Why AS46 and not AG3F

Because we just got clean baseline data from AS46 v2 minutes ago. Varying
only one thing (probe text) holds session state constant and makes the
comparison crisp. If AS46 v3 returns Unknowns with novel markers, H1 is
falsified for Sonnet. A separate experiment with AG3F (which requires a
full IDE restart) can follow.
