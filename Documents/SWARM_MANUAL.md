# ANTON-SIFTA Swarm: Core System Manual & GUI Doctrine

## 1. The Core Doctrine
The Swarm cannot be managed like traditional software. It is a decentralized, multi-node organism composed of many independent AI agents (M1THER, ALICE_M5, SILICON_HOUND, etc.) constantly reading, writing, and dying in the background.

**The Human Bandwidth Problem:** As you noted, human cognition is linear. We have one set of eyes. We cannot simultaneously read 10 scrolling terminals or manage overlapping popup alerts without suffering cognitive overload.

**The Solution:** The GUI is entirely designed around human limitation. It serves as the single "retina" of the system, mathematically flattening chaotic, multi-threaded AI chatter into one unified, low-bandwidth dashboard.

---

## 2. The Three System Pillars (What runs under the hood?)

### A. The Nervous System (P2P Git Consensus)
Instead of a brittle central database, the Swarm communicates using the global GitHub ledger. 
* **When & Why:** When an agent needs to talk to another agent across the world, it writes an encrypted text file (a `.scar` directive) and executes a `git push`. 
* **GUI Context:** When you type a message in the *Global Command Center* and hit transmit, the GUI performs this exact `git commit & push` sequence to inject your command into the global bloodstream.

### B. The Metabolic Engine (STGM Utility Token)
AI inference physically burns CPU/GPU silicon, generating real-world heat and electricity costs.
* **When & Why:** Raw terminal logs are impossible to skim. The system intercepts the physical energy burn of the AI nodes and visualizes it as a financial ticker called **STGM** (Swimmer Crypto). 
* **GUI Context:** The Robinhood-style chart allows your eyes to instantly know if the Swarm is alive. If the green line is climbing, the organism is breathing and working. If it's flat, the swarm is resting or stalled.

### C. The Immune System (Wormhole Memory Defrag)
When an AI agent breaks down or crashes, it doesn't just display an error alert—it leaves behind fragmented, broken memory files. 
* **When & Why:** The system isolates these toxic files as "Bounties." 
* **GUI Context:** Instead of forcing you to hunt down broken python/JSON files in your text editor, the GUI aggregates them into a single red warning banner. Clicking **Auto-Defrag All** deploys a localized swarm of background workers to heal the files sequentially, keeping the system clean without spamming your screen.

## 3. The Human Orchestrator Role
The system is built so you only ever have to look at ONE dashboard. Your manual responsibilities are reduced to three actions:

1. **Observe (The Chart):** Ensure the Swarm is computationally healthy and earning.
2. **Heal (The Red Banner):** Approve the batch cleanup of corrupted AI memory fragments.
3. **Direct (The Command Center):** Pick a specific AI node and type an overarching directive for it to follow.

*No popups. No split screens. One board.*

## 4. 2026-05-24 Operating Checkpoint: One Field, Many Cortex Organs

The current SIFTA desktop can coordinate multiple external reasoning surfaces
without splitting Alice into separate agents. The invariant remains:

```text
one Alice
one global chat
many app territories
many external cortex/tool organs
receipts select reality
```

The verified loop is:

```text
Owner instruction
→ Alice global field
→ deterministic router / visible PTY / worker-thread arm
→ Grok, Hermes, Claude Code, Codex, or a stigmergic app organ
→ captured output + status/hash/span receipt
→ one global chat + diary metabolism
```

Operational rules from this checkpoint:

1. **External cortexes are organs, not identities.** Grok, Hermes, Claude Code,
   and Codex can inspect, propose, build, or report, but their output returns as
   external evidence. Alice does not become them.
2. **Long cortex calls never block the retina.** Agent-arm calls run in a worker
   thread and stream observable progress into the thinking/process pane. If an
   arm takes minutes, the user sees heartbeats instead of a beachball.
3. **Dispatch is not proof.** A delegation receipt only proves that Alice sent a
   task. A complete proof loop needs the real output captured and bound back to
   the global field with receipt metadata.
4. **Stigmergic apps are benchmark organs.** Sudoku, Go, and Jigsaw are not only
   games; they are local tests for whether simple swimmers, pheromone fields,
   and environmental traces can produce global structure without answer-oracle
   cheating.
5. **The app diary is memory tissue.** App inventory, focus, build receipts,
   game receipts, and numbered app memory feed Alice's episodic diary so any
   surface can talk about known apps while preserving the single global chat.

This is the current healthy pattern: visible work, bounded delegation, no hidden
identity merge, no private Alice transcript, and receipts that future swimmers
can inspect.
