# Product Context — Why Alice Exists

## Problem
Current AI tools are stateless between sessions, hallucinate, collide when multiple doctors work on the same codebase, and have no persistent identity or economy tied to real hardware.

## Solution
A single embodied organism (Alice) that:
- Lives on one physical machine
- Has persistent structured memory (Memory Bank + ledgers)
- Produces real value via a cryptographic economy (STGM)
- Can be extended by multiple doctors safely through predator gates and MANA coordination
- Has real arms (CLI hand, desktop, etc.) that the body can detect and use

## UX Goals for the Architect
- Talk to one continuous Alice across all surfaces
- See the organism's health (CU badge, metabolic state, field stress)
- Install the hand with one command (`npm install -g @anton-sifta/alice`)
- Trust that the economy is not being polluted by sandbox doctors
